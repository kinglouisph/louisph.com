Project 1

The goal of this project is to find what places in the country have the best and worst weather. It uses a database of 8.6 million weather events recorded nationwide across seven years. The importance of this data is primarily to real estate: housing in areas with better weather should become more desirable.

Links

Data: https://www.kaggle.com/datasets/sobhanmoosavi/us-weather-events Jupyter Notebook: https://github.com/kinglouisph/datamining1

Methodology

The data I used for this project is a large database of nationwide weather events from a period of seven years. The uncompressed file size is very big, making processing the data very slow. This was not at all helped by the fact that I was using python.

The first thing I did was create a new table with the sums of results of previous tables. The new table contained total amounts of rainfall, snowfall, etc, and the lengths of time (in seconds) affected by rain, snow, etc.

This list lists the data points by airport. Airports are used because city and zip code contain many nulls, and Airports were the smallest "unit". averaging scores by county was another option but that felt like blurring the data for no reason, since counties could just be pulled from the airports.

The next thing I did was use this list to find the maxima and minima in each catigory. This was very straightforward with the new list. Note that the below values are per year (tolals / 7), and the days of precipitation represent contiguous days of weather, not the number of days that the weather occurred. That number is likely double or triple the amount shown.

These results look reasonable. The KJAX rainfall numbers may look high but it's in florida, so that's to be expected. The reason I hypothesize this is more accurate than the 25 percentile results is that it seems much easier to underestimate weather events by simply not recording them than it is to overestimate. The errors used to overestivate (misreading measurements, faulty equipment, etc) also are more likely to work both ways. Still, the 95 percentile data looked suspicious so I used 90 percintile instead.
